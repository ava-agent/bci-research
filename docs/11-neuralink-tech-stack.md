## 2. 各层技术栈详解

### 2.1 芯片层（N1 ASIC）

| 维度 | 规格 |
|------|------|
| **制程** | 65nm CMOS |
| **尺寸** | 5 × 4 mm² |
| **通道数** | 1,024（4 模块 × 256 通道） |
| **采样率** | 19.3 kHz / 10-bit SAR ADC |
| **功耗** | 24.7 mW 总功耗，5.2 μW/模拟像素 |
| **噪声** | 6.8 μVrms（LFP 模式）、8.98 μVrms（AP 模式） |
| **增益** | 43-60 dB 可配置 |
| **设计语言** | Verilog/VHDL（数字逻辑），定制模拟电路设计 |
| **EDA 工具** | AutoDesk Fusion 360（已确认使用） |

**核心创新 — 片上尖峰检测：**
```
原理: 非线性能量算子 (NEO)
公式: Ψ[x(n)] = x²(n-1) - x(n) · x(n-2)

特点:
  · 在 FPGA 硬件上执行
  · 900 纳秒超低延迟
  · 数据压缩比 200x ~ 1250x
  · 保留 >99.7% 准确率
  · 可区分同一电极上的不同神经元（基于波形形状）
```

### 2.2 嵌入式固件层

| 维度 | 技术选择 |
|------|---------|
| **语言** | C / C++（嵌入式标准）+ Rust（招聘中明确要求） |
| **OS** | 定制 RTOS 或裸机（bare-metal），安全关键系统 |
| **核心职责** | 实时数据采集、尖峰排序、无线传输协议、固件 OTA 更新 |
| **安全要求** | 医疗级故障容错设计、HIPAA 合规 |

**招聘信息揭示的关键要求：**
- 嵌入式系统 + 实时数据管线
- 内存管理、并发、高效 I/O
- 安全关键系统的软件架构
- 硬件-软件交互经验

### 2.3 信号处理 + ML 解码层

| 维度 | 技术 |
|------|------|
| **语言** | Python（模型训练/仿真）、C++（实时推理）、Rust（安全关键模块） |
| **信号处理** | 尖峰排序（Spike Sorting）、带通滤波、伪迹去除 |
| **ML 模型** | 神经网络（将神经模式映射到动作意图） |
| **训练数据** | 动物实验 + 早期人体试验数据 |
| **推理引擎** | 边缘推理框架（在植入体或 Link 设备上低功耗运行） |
| **数据管线** | 高吞吐量神经数据管道，schema 设计 + 性能优化 |

**信号处理管线：**
```
原始神经信号 (1024 通道 × 19.3kHz)
  → 模拟放大 (LNA + VGA, 43-60dB)
  → ADC 数字化 (16x SAR ADC, 10-bit)
  → 片上尖峰检测 + 压缩 (NEO, 200x-1250x)
  → 无线传输 → Link 设备
  → 尖峰排序 (分类不同神经元)
  → 特征提取 (firing rate, 模式)
  → ML 解码 (神经网络: 神经模式 → 运动意图)
  → 意图输出 (光标位置 / 键盘输入 / 语音)
```

## 5. 关键技术洞察

### 5.1 片上压缩是系统瓶颈突破

1,024 通道 × 19.3kHz × 10bit = 约 200Mbps 的原始数据，不可能全部无线传输。片上 NEO 尖峰检测算法在 900ns 内完成 200x-1250x 的压缩，只传输有意义的 Spike 事件——这是 Neuralink 能做到无线实时 BCI 的根本原因。

### 5.2 Rust 在安全关键固件中的引入

在传统嵌入式领域 C/C++ 几乎是唯一选择，但 Neuralink 在安全关键的脑植入设备上引入 Rust 的内存安全保证——这说明他们对代码安全性的要求极高。

### 5.3 完全封闭的技术生态

Neuralink 的技术栈全部自研、不对外开放 SDK。外部开发者无法接入 Neuralink 的设备或数据。Neuralink 解决的是"侵入式高精度解码"问题，消费级 BCI 应用需要走非侵入式 + 开源 SDK（BrainFlow）路线。
